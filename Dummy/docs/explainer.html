2 · AXI bus widths

BusDirectionDataAddrPurpose
S_AXIL (AXI4-Lite slave)VEGA → accel32-bit8-bitWrite START + descriptor pointers, read STATUS
M_AXI (AXI4 master)accel ↔ DDR3128-bit32-bitBurst read weights/activations, burst write feature maps + detections
Neither is 64-bit. Slave = 32-bit (control only). Master = 128-bit (16 B/beat, max DDR throughput). Set in accel_pkg.sv (AXIL_DATA_W, AXI_DATA_W). For a 64-bit master, set AXI_DATA_W=64.

3 · Data flow (Option B)

The accelerator runs the whole network — backbone + neck + head — layer by layer through the same GEMM engine, then writes all detections to DDR. The CPU reads that tensor and picks the winner in software. The image comes from the CPU into DDR; nothing image-sized touches UART.

5 · Why Option B — parallelism

model2vec needs only the prompt; YOLO needs only the image. They are independent except for the final score. Option B returns all detections to DDR so the CPU runs model2vec fully in parallel and scores at the end — no ordering prerequisite. Toggle to compare:

6 · One inference, start to finish

#WhoAction
0VEGAPut image into DDR. Start model2vec (prompt → ranked classes) — runs in parallel with everything below.
1VEGABuild descriptor list in DDR, write DESC_BASE/COUNT, pulse START.
2ACCELSelf-walk all layers: read_master → mac_tile GEMM → vector_unit → write_master, layer after layer.
3ACCELLast layer writes the full detection tensor to DDR. Raise one irq.
4VEGARead detections from DDR, apply score = conf/(rank+1) with model2vec ranks, pick best (or top-k).
5VEGADone. UART optionally echoes status to a host for a functionality check.
CPU↔accel control traffic = 1 START write + 1 IRQ + a few STATUS reads. Detections move over AXI ↔ DDR; the CPU reads them from DDR, not through registers.
